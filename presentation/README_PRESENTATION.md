# presentation 目录说明

本目录用于保存软件开发课程答辩与展示相关材料说明，包括课程版 PPT、答辩讲稿、展示提纲和 PPT 修改记录。

## 建议内容

- `outline.md`：答辩展示大纲，说明每一页 PPT 的主要内容。
- `speech_script.md`：答辩讲稿或演示串词。
- `revision_notes.md`：PPT 修改记录、教师反馈和后续调整说明。
- `slides_link.md`：如 PPT 文件较大，可在此记录网盘、课程平台或附件提交位置。

## PPT 内容建议

课程答辩 PPT 可按以下顺序组织：

1. 项目背景与问题定义。
2. 项目目标与核心功能。
3. 系统总体架构。
4. 后端、前端、模型、数据库模块说明。
5. DTI 预测与个性化推荐流程。
6. 关键页面与运行截图。
7. 测试结果与部署情况。
8. 小组分工、总结与后续改进。

## 提交注意事项

- 如果 GitHub 仓库继续使用当前 `.gitignore`，`.ppt` 和 `.pptx` 文件默认不会提交。
- 建议在本目录保留 PPT 链接、讲稿和修改说明；正式 PPT 可按课程要求通过课程平台或压缩包提交。
- PPT 中不要展示真实数据库密码、服务器账号、API Key、Token 等敏感信息。
- 如果确实需要将 PPT 纳入 GitHub，需要先确认文件大小和课程提交要求，再调整 `.gitignore`。
