# figures 目录说明

本目录用于保存软件开发课程文档中引用的图表资源。

## 建议图表

- `architecture.png`：系统总体架构图。
- `use_case_diagram.png`：用户用例图。
- `sequence_prediction.png`：DTI 预测或推荐流程时序图。
- `er_diagram.png`：数据库 ER 图。
- `workflow_recommendation.png`：个性化药物推荐流程图。
- `deployment_diagram.png`：部署结构图。

## 命名建议

建议使用英文小写、下划线和清晰语义命名，避免使用截图自动生成的随机文件名。

## 引用示例

在 `docs/` 下的 Markdown 文档中引用：

```md
![系统架构图](figures/architecture.png)
```

## 注意事项

- 推荐提交 `.png`、`.jpg`、`.jpeg`、`.svg` 格式。
- 图中不要展示真实密码、API Key、服务器账号等敏感信息。
- 如果图表来自外部工具导出，建议同时保留可编辑源文件在课程压缩包中，GitHub 中提交图片版本即可。
